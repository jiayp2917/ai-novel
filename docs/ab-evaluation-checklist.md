# A/B 评审检查表（4 维评分）

> 配套 `docs/ui-refactor-plan.md` §6.4 / §8。每张 AI 生成的资产、每组 A/B 截图都必须走本表。

## 0. 硬门槛（不通过即弃选，不计分）

- [ ] **OCR 扫文字**：本地 tesseract 扫描整图，可读字符数 = 0。
- [ ] **无 logo / UI 假象**：不包含可识别的 logo、品牌名、按钮形态、输入框、列表项。
- [ ] **无人物 / 动物**：不包含具象角色（人形、动物、漫画角色）。
- [ ] **无敏感内容**：不含真实作品片段、API key、私密路径、用户姓名。
- [ ] **尺寸合规**：与 `assets/theme/README.md` §3 表的尺寸一致。

## 1. 可读性（Readability）— 1-5 分

> 关键：图与文字的对比度、装饰稀疏度、是否会与前景文字竞争注意力。

- 5：与 `--ink` / `--paper-ink` 文字对比度 ≥ 7:1，装饰可忽略，前景文字压倒性清晰。
- 4：对比度 4.5-7:1（即 WCAG AA 起步），装饰不抢戏。
- 3：对比度 3-4.5:1，文字可读但有压迫感。
- 2：对比度 < 3:1，需要 hover/active 状态时易读性下降。
- 1：装饰物明显压过文字，关键信息无法识别。

## 2. 一致性（Consistency）— 1-5 分

> 同主题内 6 张图（bg/paper/dialog/chip/divider/button）的风格连贯度。

- 5：6 张图明显属于同一作者 / 同一批次；色相、纹理密度、笔触风格统一。
- 4：4-5 张图明显一致，1-2 张有轻微风格偏离。
- 3：风格可辨但有 2-3 张明显不同。
- 2：3 张以上风格不连贯。
- 1：明显拼凑感。

跨主题检查：breeze / stargold / silk 三主题的 `bg.jpg` 色相必须明确区分（暖纸 / 深星 / 珍珠白），不允许两张主题的 bg 在第一眼无法分辨。

## 3. 情绪（Mood）— 1-5 分

> 与主题名「清风稿纸 / 星空鎏金 / 白丝质感」的情绪匹配度。

- 5：一眼能识别出主题名对应的氛围；与产品定位（长篇小说编辑器）契合。
- 4：氛围明确，但与产品定位略偏。
- 3：氛围可辨但不够突出。
- 2：氛围模糊或与产品定位冲突。
- 1：情绪明显不符。

## 4. 噪点（Noise）— 1-5 分（高分 = 低噪点）

- 5：无拉伸、无拼接缝、无重复图块边界、人物残影、UI 假象。
- 4：放大 200% 后能看见 1 处轻微瑕疵（如某点颗粒过粗），正常尺寸下不可见。
- 3：1-2 处明显瑕疵，桌面端可见。
- 2：3 处以上瑕疵或 1 处严重影响画面（如拼接缝、对角线拉伸）。
- 1：明显低质，破坏产品感。

## 弃选规则

- 任意维度 ≤ 2 分 → 弃选。
- 总分 < 14/20 → 弃选。
- 同主题内 6 张图一致性 < 3 → 整组重生成。

## 复审节奏

- 每次重生成后立即评分，不允许「先攒图再打分」。
- A/B 截图存到 `runtime/ab-screenshots/<phase>/<view>/{solid,ai}.png`，文件名后缀明确。
- Phase 1 末必须出「首页 / 写作 / AI 工作台 / 设置」× 三主题 × solid/ai = 24 张对照。
