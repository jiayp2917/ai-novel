# 小说编辑器开发手册

## 核心原则

- 代码仓库和小说工作区必须保持边界清晰。
- 模型输出先进入 artifact。
- 正文写回只允许发布门执行。
- 设定和章纲默认只生成提案，不自动覆盖。
- reviewer 只审核，不修文、不写文。
- 无证据问题必须转 `admin` 或人工处理。
- 不读取、不打印、不提交 `key.txt`、`.env` 或真实 API key。

## 后端分层

```text
backend/app/api/       # FastAPI 路由
backend/app/services/  # 业务服务
backend/app/services/pipeline/ # 自动流水线
backend/app/db/        # SQLAlchemy 模型与会话
backend/tools/         # 本机工具和验收脚本
```

工作流应保持编排职责，不把 prompt、路径细节、发布细节全部塞进状态机。

## 前端分层

```text
frontend/src/pages/       # 页面
frontend/src/components/  # 组件
frontend/src/store*.ts    # Zustand 状态和持久化
frontend/e2e/             # Playwright 测试
```

写作界面优先保证正文区域无遮挡、右键批注可用、目录清晰、侧栏可隐藏。模型调用、任务、成本统计应和正文编辑分离。

## 发布门规则

发布正文前必须满足：

1. artifact 存在且 hash 匹配。
2. artifact 绑定章节和源文件。
3. 最新 review 存在。
4. review 的 candidate/source/version hash 与 artifact 一致。
5. source kind 是 `chapters`。
6. 人工确认发布。
7. 写入前生成 backup 和 diff。
8. 写入后重新扫描并更新短记忆。

## 模型岗位

默认岗位：

- `reviewer`：DeepSeek V4 Pro
- `writer` / `quick_fix` / `fixer`：Kimi K2.6
- `memory` / `long_context`：Qwen 3.6 Plus
- `structural_fix`：GLM 5.1
- `outliner` / `arbiter`：Qwen 3.6 Max Preview

不要因为一次探测成功就改默认路由。路由调整需要样本、成功率、证据率、复审通过率、耗时和成本数据支撑。

## 测试矩阵

常规提交前：

```powershell
python -m compileall -q .\backend .\tests
python -m pytest -q
cd .\frontend
npm run build
```

前端交互变化：

```powershell
cd .\frontend
npm run e2e
```

发布门或流水线变化：

```powershell
python -m backend.tools.sandbox_publish_smoke
python -m backend.tools.sandbox_pipeline_smoke --workspace runtime/sandbox_pipeline_workspace --chapters 3 --reset
```

模型治理变化：

```powershell
python -m backend.tools.model_usage_report --out runtime\logs\model_usage_report.md
```

## 打包边界

系统包只包含：

- `backend/`
- `frontend/`
- `config/`
- `content/` 空模板占位文件
- `skills/`
- `tests/`
- `docs/`
- 项目文档和依赖文件

系统包不包含：

- `00-系统/`
- `01-设定/`
- `02-正文/`
- `03-章纲/`
- `runtime/`
- `content/settings/`、`content/outlines/`、`content/chapters/` 下的真实素材
- `key.txt`
- `.env`
- 缓存和构建产物
