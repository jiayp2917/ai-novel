# 小说编辑器用户手册

## 适用对象

本手册面向写作者。你只需要会打开网页、选择目录、编辑文字和点击按钮。

## 启动

1. 启动后端：

```powershell
cd D:\2917\numeric-monster
python -m backend.tools.run_backend_with_keys
```

2. 启动前端：

```powershell
cd D:\2917\numeric-monster\frontend
npm run dev
```

3. 打开前端：

```text
http://127.0.0.1:5173
```

## 日常写作流程

1. 在“设置/模型”里确认当前小说目录，或从作品列表打开最近作品。
2. 点击“重新扫描”，让系统读取设定、章纲、正文。
3. 进入“写作”，从左侧目录打开章节。
4. 直接编辑正文草稿，或右键打开高频操作菜单。
5. 需要批注时，拖选文字后右键新建批注。
6. 点击“保存草稿”，系统只写入 `runtime/artifacts/`，不会直接覆盖正文。
7. 如果是你手工改的草稿，可以查看改动后确认写回；系统仍会先校验、生成 diff 和备份。
8. 如果是 AI 生成或 AI 修订的草稿，先进入“AI 工作台”检查，通过后再查看改动和确认写回。
9. 设定和章纲默认只生成提案，不自动覆盖。

## 页面怎么分工

- 首页：查看当前作品状态和常用入口。
- 写作：打开章节、编辑草稿、搜索、批注、保存草稿、查看版本。
- 资料库：查看系统设定、小说设定和章纲。
- AI 工作台：检查草稿、AI 修订、模型调用、质量趋势、上下文裁剪和 skills 状态。
- 自动流水线：批量生成、检查和修订章节草稿。
- 设置/模型：作品列表、工作区切换、模型路由和运行状态。

## 安全边界

- AI 输出先保存为草稿/候选，不直接写正文。
- 正文写回只通过发布门。
- 发布前会生成备份和 diff。
- 人工草稿不强制 AI 检查；AI 草稿必须检查通过。
- 设定、章纲问题默认转人工确认。
- 无证据的审核问题不会自动进入修复。

## 常见问题

### 看不到章节

检查工作区目录是否包含以下旧布局之一：

```text
00-系统/
01-设定/
02-正文/
03-章纲/
```

或新布局：

```text
content/settings/
content/outlines/
content/chapters/
```

然后重新扫描。

### 模型调用失败

进入“AI 工作台”或“设置/模型”查看模型调用记录。常见原因包括 key 未加载、预算暂停、网络失败、供应商返回错误。

### 质量趋势显示“数据不足”

这表示当前样本太少，系统不能判断某个 AI 岗位是否稳定。继续使用 AI 检查、写作或修订后，才会出现证据率、字数达标率、复审通过率等数据。

### 不想改真实正文

只生成候选和审核，不点击发布门确认。沙盒发布测试只操作 `runtime/sandbox_workspace`。
