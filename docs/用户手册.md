# 小说编辑器用户手册

## 适用对象

本手册面向写作者。你只需要会打开网页、选择目录、编辑文字和点击按钮。

本工具是通用长篇小说编辑器，不绑定某一本书。一个作品对应一个工作区，作品自己的设定、风格、章纲和记忆都应放在该作品工作区中。

## 启动

1. 启动后端：

```powershell
cd <repo-root>
python -m backend.tools.run_backend_with_keys
```

2. 启动前端：

```powershell
cd <repo-root>\frontend
npm run dev
```

3. 打开前端：

```text
http://127.0.0.1:5173
```

## 两种日常流程

### AI 主导生产流

适合从构思、设定、大纲、章纲推进到正文初稿。

1. 在工作区准备作品档案、设定或大纲。
2. 如果信息不完整，AI 会生成补全提案，或通过弹窗向你询问缺失信息。
3. 设定、大纲、卷纲、章纲都先生成提案，你确认后才会成为正式资料。
4. 正文生成前先形成单章写作卡，用来固定本章目标、场景、人物、冲突、钩子和禁写项。
5. AI 生成的正文先进入草稿候选，不直接写入正式正文。
6. 草稿需要检查、查看改动，再由你确认写回。
7. 发布后系统会更新短记忆；风格学习只根据 AI 初稿和你确认后的修改差异生成提案。

### 人工主导编辑流

1. 在“设置/模型”确认当前小说目录，或从顶部当前工作区入口进入管理页。
2. 点击“重新扫描”，让系统读取设定、章纲和正文。
3. 进入“写作”，从左侧目录打开章节。
4. 直接编辑正文，或拖选文字后右键创建批注。
5. 点击“保存正文版本”。系统会记录一个可预览、可对比、可发布的正文版本，不会立刻覆盖正式正文。
6. 在右侧“版本”里预览改动；确认无误后发布该版本。
7. AI 生成或 AI 修订的内容必须先进入“AI 工作台”检查，通过后才能查看改动并确认写回。
8. 系统设定、小说设定和章纲默认只生成提案，不自动覆盖源文件。

人工编辑不依赖模型可用性。即使模型不可用，你仍然可以写作、保存正文版本、查看 diff 并发布手写版本。

## 页面怎么分工

- 首页：查看当前作品、最近章节、待处理事项和快捷入口。
- 写作：打开章节、编辑正文、搜索、批注、保存正文版本、查看版本和记忆。
- AI 素材库：只读查看系统设定、小说设定和章纲，生成提案并查看对比；不展示正文，也不提供正文写回入口。
- AI 工作台：选择草稿、检查草稿、按批注修订、查看改动、确认写回和查看上下文。
- 自动流水线：批量生成、检查和修订章节草稿，优先用于 dry-run 和运行状态观察。
- 设置/模型：管理工作区、模型连通、质量趋势、调用用量、skills 和高级日志。

## 安全边界

- AI 输出先保存为草稿、提案或正文版本，不直接写正文。
- 正文写回只通过发布门。
- 发布前会生成备份和 diff。
- 人工正文版本不强制 AI 检查；AI 草稿必须检查通过。
- 设定和章纲默认走提案流程。
- 作品档案、长期记忆补充和风格规则也必须先生成提案，你确认后才会成为正式资料。
- 强制发布也不能绕过 diff、备份、人工确认和发布记录。
- 无证据的审核问题不会自动进入修复。
- 未发布草稿不会进入长期记忆，也不会参与风格学习。

## 常见问题

### 看不到章节

检查工作区目录是否包含以下旧布局之一：

```text
00-系统/
01-设定/
02-正文/
03-章纲/
```

或新布局：

```text
00-设定/
01-大纲/
02-正文/
03-章纲/
```

或 content 布局：

```text
content/settings/
content/outlines/
content/chapters/
```

然后重新扫描。正文文件需要标准章节标题才会进入章节目录；非标准 Markdown 会在写作页的未识别正文文件区域提示处理方式。

### 模型调用失败

进入“AI 工作台”或“设置/模型”查看失败原因。当前默认 AI 路径是 Agnes AI；常见原因包括 `AGNES_API_KEY` 未加载、预算暂停、网络失败、供应商返回错误。

### 质量趋势显示“数据不足”

这表示当前样本太少，系统不能判断某个 AI 岗位是否稳定。继续使用 AI 检查、写作或修订后，才会出现证据率、字数达标率、复审通过率等数据。

### 不想改真实正文

只生成正文版本、草稿或提案，不点击发布确认。沙盒发布测试只操作 `runtime/sandbox_workspace`。

本机示例内容可以从 `runtime/sandbox_workspace` 写回到项目 `content/` 目录用于手测；这些内容被 `.gitignore` 忽略，不会随系统代码提交。真实作品建议放在仓库外部的小说工作区。
