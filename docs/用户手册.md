# 小说编辑器用户手册

## 适用对象

本手册面向写作者。你只需要会打开网页、选择目录、编辑文字和点击按钮。

## 启动

1. 启动后端：

```powershell
cd D:\2917\numeric-monster
python -m backend.tools.run_backend_with_keys
```

2. 启动前端：

```powershell
cd D:\2917\numeric-monster\frontend
npm run dev
```

3. 打开前端：

```text
http://127.0.0.1:5173
```

## 日常写作流程

1. 在“工作区”确认当前小说目录。
2. 点击“重新扫描”，让系统读取设定、章纲、正文。
3. 进入“正文编写”，从左侧目录打开章节。
4. 直接编辑正文草稿，或右键打开高频操作菜单。
5. 需要批注时，拖选文字后右键新建批注。
6. 保存候选时，系统只写入 `runtime/artifacts/`，不会直接覆盖正文。
7. 候选必须经过审核和 diff 查看后，才能进入发布门。
8. 正文发布必须人工确认；设定和章纲默认只生成提案，不自动覆盖。

## 安全边界

- AI 输出先保存为候选，不直接写正文。
- 正文写回只通过发布门。
- 发布前会生成备份和 diff。
- 设定、章纲问题默认转人工确认。
- 无证据的审核问题不会自动进入修复。

## 常见问题

### 看不到章节

检查工作区目录是否包含以下旧布局之一：

```text
00-系统/
01-设定/
02-正文/
03-章纲/
```

或新布局：

```text
content/settings/
content/outlines/
content/chapters/
```

然后重新扫描。

### 模型调用失败

进入“模型与任务”查看模型调用记录。常见原因包括 key 未加载、预算暂停、网络失败、供应商返回错误。

### 不想改真实正文

只生成候选和审核，不点击发布门确认。沙盒发布测试只操作 `runtime/sandbox_workspace`。
