# 小说编辑器界面模块与功能串联

本文按当前前端实际界面梳理，不按历史设想描述。目标是让后续开发和使用时清楚：每个界面负责什么、不能做什么，以及功能如何串成完整流程。

## 1. 模块总览

| 界面 | 定位 | 核心功能 | 不承担的功能 |
| --- | --- | --- | --- |
| 首页工作台 | 入口和项目概览 | 查看源文件数、章节数、今日调用、运行任务；进入写作、资料库、AI 工作台、自动流水线 | 不直接编辑正文，不直接调用模型 |
| 写作 | 作者日常主界面 | 目录浏览、正文阅读/编辑、搜索跳转、右键批注、保存草稿、查看版本和记忆 | 不做 AI 审核、差异发布、模型探测 |
| 设定/章纲 | 资料库与提案入口 | 阅读系统设定、小说设定、章纲；为设定/章纲生成提案 | 不通过普通发布门覆盖设定/章纲源文件 |
| AI 工作台 | 草稿检查、AI 修订和写回确认 | 上下文预览、生成审核快照、按批注生成候选、运行任务、检查草稿、查看改动、确认写回 | 不替代人工写作主界面 |
| 自动流水线 | 批量章节流程 | 按章节范围批量生成、检查、修订；支持 dry-run、暂停、恢复、重试 | 不绕过发布门，不默认覆盖真实正文 |
| 设置/模型 | 作品管理和高级观测 | 作品列表、最近打开、打开并扫描、整理记忆库、模型分工、连通测试、成本/质量趋势、skills、事件 | 不承载正文编辑体验 |
| 状态面板 | 全局反馈 | 显示最近操作、任务执行、错误和成功提示 | 不作为主要业务入口 |

## 2. 当前主导航结构

当前主导航是：

```text
首页 -> 写作 -> 资料库 -> AI 工作台 -> 自动流水线 -> 设置/模型
```

其中“作品/工作区”不是单独放在主导航里的一级入口，当前主要出现在：

- 首页“管理作品”入口；
- 设置/模型里的“作品列表 / 最近打开”；
- 内部仍保留 `workspace` 视图能力，后续如需要可以重新放回导航。

## 3. 各界面功能边界

### 3.1 首页工作台

用途：

- 给用户一个低负担入口，先判断当前作品是否已扫描。
- 展示源文件、正文、今日 AI 调用、运行任务等概览。
- 提供“进入写作”“打开 AI 工作台”“进入资料库”“打开自动流水线”等快捷入口。

串联关系：

- 第一次使用：进入首页 -> 管理作品 -> 打开并扫描 -> 进入写作。
- 日常使用：进入首页 -> 直接进入写作或 AI 工作台。

### 3.2 写作

用途：

- 这是作者人工创作的主界面。
- 左侧目录用于打开系统设定、小说设定、章纲和正文分卷。
- 中间区域用于阅读、搜索、跳章、编辑草稿、保存草稿。
- 右侧栏只保留“批注 / 版本 / 记忆”。

主要动作：

- 打开章节；
- 上一章 / 下一章 / 输入章号跳转；
- 搜索正文并跳转命中位置；
- 进入编辑模式并修改草稿；
- 右键选中文本创建批注；
- 保存草稿到 `runtime/artifacts/`；
- 查看当前章节版本、草稿、写回记录；
- 查看批注规则和短记忆摘要。

重要边界：

- 保存草稿不覆盖正式正文。
- 写作页不再承载候选审核和发布按钮。
- 需要检查、查看改动或写回时，进入 AI 工作台。

### 3.3 设定/章纲

用途：

- 阅读系统设定、小说设定、章纲。
- 对设定/章纲生成候选提案。
- 提案可以检查和对比，但不通过普通发布流程直接覆盖源文件。

串联关系：

```text
资料库选择设定/章纲
  -> 生成候选提案
  -> 查看检查结果和改动
  -> 人工决定是否采纳
```

重要边界：

- 设定/章纲默认是“提案制”，不是“直接写回制”。
- 普通正文发布门只服务正文。

### 3.4 AI 工作台

用途：

- 集中放置会影响候选、审核、差异和写回的功能。
- 让写作界面保持干净，把 AI 流程和人工写作拆开。

主要动作：

- 查看当前章节上下文预览；
- 从当前正文生成审核快照；
- 根据已勾选批注生成修订候选；
- 运行一次任务队列；
- 选择草稿；
- 检查草稿；
- 查看改动；
- 确认写回正文；
- 查看最近任务；
- 调用记忆整理入口。

人工草稿与 AI 草稿区别：

- 人工草稿：可直接查看改动并走本地校验、人工确认、发布门写回。
- AI 草稿：必须先检查，通过后才能写回。

写回固定边界：

```text
草稿存在
  -> 草稿归属当前章节
  -> 本地校验通过
  -> AI 草稿已检查通过
  -> 查看改动
  -> 用户确认写回
  -> 生成备份和 diff
  -> 发布门写入正文
```

### 3.5 自动流水线

用途：

- 面向批量生产或批量检查，不是日常手写入口。
- 适合沙盒验证、1-10 章 dry-run、批量生成缺失草稿、批量检查修订。

模式：

- 只检查已有草稿；
- 只补齐缺失草稿；
- 检查并生成修订草稿；
- 全自动生成、检查、修订。

安全默认：

- 默认 `dry-run`，只生成草稿、检查结果和报告。
- 正式写回仍必须进入发布门确认。

串联关系：

```text
选择章节范围
  -> 选择模式
  -> 设置分批和修订轮次
  -> 创建自动流水线
  -> 运行队列
  -> 查看每章状态
  -> 失败/暂停时人工处理
  -> 需要写回时回到 AI 工作台/发布门
```

### 3.6 设置/模型

用途：

- 承载不适合放在写作页的高级能力。
- 包括作品管理、模型连接、模型质量、成本、上下文预算、skills 和运行事件。

作品管理：

- 最近打开；
- 添加本地作品；
- 打开并扫描；
- 重新扫描当前作品；
- 整理记忆库；
- 查看识别到的素材和正文数量。

模型任务：

- 查看 AI 分工；
- 连通测试；
- 查看待办任务；
- 继续执行任务；
- 查看最近 AI 调用；
- 查看模型质量趋势；
- 查看上下文裁剪记录；
- 查看写回记录、skills 和运行事件。

重要边界：

- 连通测试会真实调用模型，必须用户确认。
- 本地 token/usage 只是可见下限，真实消耗以供应商控制台为准。

### 3.7 状态面板

用途：

- 展示最近操作反馈，例如“保存草稿成功”“检查失败”“重新扫描完成”。
- 作为用户即时反馈，不作为核心业务导航。

当前设计：

- 收起时保持紧凑；
- 展开为浮层，不挤压正文布局；
- 适合排查刚发生的操作结果。

## 4. 主流程串联

### 4.1 首次使用流程

```text
首页
  -> 设置/模型
  -> 添加本地作品
  -> 打开并扫描
  -> 整理记忆库
  -> 写作
  -> 打开章节
```

验收点：

- 左侧目录能看到设定、章纲、正文分卷；
- 首页章节数不为 0；
- 写作页能打开章节正文。

### 4.2 人工写作流程

```text
写作
  -> 打开章节
  -> 编辑草稿
  -> 保存草稿
  -> AI 工作台
  -> 选择草稿
  -> 查看改动
  -> 确认写回正文
```

说明：

- 人工草稿不强制 AI 检查。
- 仍必须经过本地校验、改动对比、人工确认和发布门。

### 4.3 批注修订流程

```text
写作
  -> 选中文本右键创建批注
  -> 右侧批注列表勾选批注
  -> AI 工作台
  -> 按批注生成候选
  -> 运行任务队列
  -> 检查草稿
  -> 查看改动
  -> 确认写回正文
```

说明：

- 批注是修订依据。
- AI 修订产物先进入草稿/候选，不直接改正文。

### 4.4 设定/章纲提案流程

```text
资料库
  -> 选择系统设定 / 小说设定 / 章纲
  -> 生成候选提案
  -> 检查提案
  -> 查看改动
  -> 人工采纳
```

说明：

- 当前不提供普通发布按钮直接覆盖设定/章纲。
- 这是为了避免 AI 修改基础设定后影响全书一致性。

### 4.5 全自动流水线流程

```text
设置/模型
  -> 确认作品、模型和额度
  -> 自动流水线
  -> 选择章节范围和模式
  -> 保持 dry-run
  -> 创建流水线
  -> 运行队列
  -> 查看每章结果
  -> manual_required 交给人工处理
  -> 需要写回的草稿回到 AI 工作台确认
```

说明：

- 自动流水线用于批量生产和验证。
- 不应替代日常人工编辑。
- 真实工作区建议先 dry-run，沙盒验证通过后再考虑写回。

### 4.6 模型质量治理流程

```text
AI 工作台 / 自动流水线产生模型调用
  -> 设置/模型
  -> 查看质量趋势
  -> 查看上下文裁剪
  -> 查看失败原因和成本
  -> 根据数据决定是否调整模型路由
```

指标：

- AI 检查证据率；
- AI 写作字数达标率；
- AI 修订复审通过率；
- context_degraded 章节；
- 模型失败率和预算暂停。

## 5. 推荐的信息架构原则

1. 写作页只服务作者的正文编辑，不堆模型、成本和发布细节。
2. AI 工作台处理草稿检查、修订、差异和写回确认。
3. 自动流水线只做批量流程和报告，不绕过发布门。
4. 设置/模型承载高级信息，普通作者不必每天进入。
5. 设定/章纲保持提案制，避免基础资料被自动覆盖。
6. 所有写回必须有草稿、校验、改动对比、人工确认和备份。

## 6. 当前可优化点

- 主导航目前没有单独“作品/工作区”一级入口，工作区管理主要折叠在设置/模型中；如果用户频繁切换作品，可考虑重新放回主导航。
- 旧的审核、发布、记忆、模型独立页面代码已清理；当前主入口收敛到 AI 工作台、自动流水线、设置/模型。
- 写作页保存草稿后只提示去 AI 工作台，后续可增加更明显的“去查看改动”快捷跳转，但不要把发布面板重新塞回写作右侧栏。
