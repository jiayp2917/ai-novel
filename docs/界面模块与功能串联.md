# 小说编辑器界面模块与功能串联

本文按当前前端实际界面梳理。目标是让后续开发和使用时清楚：每个界面负责什么、不能做什么，以及功能如何串成完整流程。

## 1. 模块总览

| 界面 | 定位 | 核心功能 | 不承担的功能 |
| --- | --- | --- | --- |
| 首页 | 入口和项目概览 | 查看当前作品、源文件数、正文数、运行任务和快捷入口 | 不直接编辑正文，不直接发布，不直接执行模型任务 |
| 写作 | 作者日常主界面 | 目录浏览、正文阅读/编辑、搜索跳转、右键批注、保存正文版本、查看版本和记忆 | 不展示模型细节，不承载 AI 审核主流程 |
| AI 素材库 | 设定和章纲的只读素材入口 | 阅读系统设定、小说设定、章纲；生成提案；查看提案对比 | 不展示正文，不提供正文写回按钮 |
| AI 工作台 | 草稿检查、AI 修订和写回确认 | 选择草稿、检查草稿、按批注修订、查看改动、确认写回、上下文预览 | 不替代人工写作主界面 |
| 自动流水线 | 批量章节任务看板 | 按章节范围批量生成、检查、修订；支持 dry-run、暂停、恢复、重试和删除记录 | 不绕过发布门，不默认覆盖真实正文 |
| 设置/模型 | 工作区管理和高级观测 | 工作区、模型连通、质量趋势、调用用量、上下文预算、skills、事件日志 | 不承载正文编辑体验 |

## 2. 当前主导航结构

当前主导航是：

```text
首页 -> 写作 -> AI 素材库 -> AI 工作台 -> 自动流水线
```

“设置/模型”放在左侧底部入口和顶部当前工作区入口中。顶部当前工作区可点击进入工作区管理区域。

## 3. 各界面功能边界

### 3.1 首页

用途：

- 给用户一个低负担入口，先判断当前作品是否已扫描。
- 展示源文件、正文、运行任务和待处理事项。
- 提供“进入写作”“打开 AI 工作台”“进入 AI 素材库”“打开自动流水线”等快捷入口。

串联关系：

```text
第一次使用：首页 -> 设置/模型 -> 添加或打开作品 -> 扫描 -> 写作
日常使用：首页 -> 写作 / AI 工作台 / 自动流水线
```

### 3.2 写作

用途：

- 这是作者人工创作的主界面。
- 左侧目录用于打开系统设定、小说设定、章纲和正文分卷；在窄屏下作为抽屉打开。
- 中间区域用于阅读、搜索、跳章、编辑正文和保存正文版本。
- 右侧栏只保留“批注 / 版本 / 记忆”。

主要动作：

- 打开章节；
- 上一章 / 下一章 / 输入章号跳转；
- 搜索正文并跳转命中位置；
- 进入编辑模式并修改正文；
- 拖选文本后右键创建批注；
- 保存正文版本；
- 预览历史版本、查看改动、确认发布；
- 查看批注规则和短记忆摘要。

重要边界：

- 保存正文版本不立刻覆盖正式正文。
- 人工正文版本可以不经 AI 检查，但仍要通过本地校验、改动对比、人工确认和备份。
- 写作页常态不展示模型、产物编号、校验值、原始 JSON 等排错信息。

### 3.3 AI 素材库

用途：

- 阅读系统设定、小说设定和章纲。
- 对设定和章纲生成提案。
- 查看提案对比和人工采纳说明。

串联关系：

```text
AI 素材库
  -> 选择系统设定 / 小说设定 / 章纲
  -> 查看只读内容
  -> 生成提案
  -> 查看提案与改动
  -> 人工决定是否采纳
```

重要边界：

- AI 素材库不展示正文目录。
- 设定和章纲默认是提案制，不通过正文发布门直接覆盖源文件。

### 3.4 AI 工作台

用途：

- 集中处理会影响草稿、检查、差异和写回的 AI 流程。
- 让写作界面保持干净，把 AI 辅助流程和人工写作拆开。

主要动作：

- 查看当前章节上下文预览；
- 按批注生成修订草稿；
- 在高级操作中创建待检查副本；
- 继续处理队列；
- 选择草稿；
- 检查草稿；
- 查看改动；
- 确认写回正文；
- 查看最近任务和失败原因。

人工正文版本与 AI 草稿区别：

- 人工正文版本：可直接查看改动并走本地校验、人工确认、发布门写回。
- AI 草稿：必须先检查，通过后才能写回。

写回固定边界：

```text
草稿存在
  -> 草稿归属当前章节
  -> 本地校验通过
  -> AI 草稿已检查通过
  -> 查看改动
  -> 用户确认写回
  -> 生成备份和 diff
  -> 发布门写入正文
```

工程字段处理：

- 草稿编号、产物路径、校验值、模型记录和原始检查 JSON 只放在“高级详情 / 排错信息”中。
- 普通作者主流程只看到“选择草稿、检查草稿、查看改动、确认写回”。

### 3.5 自动流水线

用途：

- 面向批量生产或批量检查，不是日常手写入口。
- 适合沙盒验证、少量章节 dry-run、批量生成缺失草稿、批量检查修订。

模式：

- 只检查已有草稿；
- 只补齐缺失草稿；
- 检查并生成修订草稿；
- 自动生成、检查和修订。

安全默认：

- 默认 dry-run，先生成草稿、检查结果和报告。
- 正式写回仍必须进入发布门确认。

串联关系：

```text
选择章节范围
  -> 选择模式
  -> 设置分批和修订轮次
  -> 创建自动流水线
  -> 运行队列
  -> 查看每章状态
  -> 失败或暂停时人工处理
  -> 需要写回时回到 AI 工作台或版本发布门
```

### 3.6 设置/模型

用途：

- 承载不适合放在写作页的高级能力。
- 包括工作区管理、模型连接、模型质量、调用用量、上下文预算、skills 和运行事件。

作品管理：

- 最近打开；
- 添加本地作品；
- 打开并扫描；
- 重新扫描当前作品；
- 整理记忆库；
- 查看识别到的素材和正文数量。

模型任务：

- 查看 AI 分工；
- 连通测试；
- 查看待办任务；
- 继续执行任务；
- 查看最近 AI 调用；
- 查看模型质量趋势；
- 查看上下文裁剪记录；
- 查看写回记录、skills 和运行事件。

重要边界：

- 连通测试会真实调用模型，必须用户确认。
- 本地 token/usage 只是可见下限，真实消耗以供应商控制台为准。
- provider、model、base_url、原始 JSON 等信息应分层放在高级详情中。

## 4. 主流程串联

### 4.1 首次使用流程

```text
首页
  -> 设置/模型
  -> 添加本地作品
  -> 打开并扫描
  -> 整理记忆库
  -> 写作
  -> 打开章节
```

验收点：

- 左侧目录能看到设定、章纲、正文分卷；
- 首页章节数不为 0；
- 写作页能打开章节正文。

### 4.2 人工写作流程

```text
写作
  -> 打开章节
  -> 编辑正文
  -> 保存正文版本
  -> 预览版本
  -> 查看改动
  -> 确认发布版本
```

说明：

- 人工正文版本不强制 AI 检查。
- 仍必须经过本地校验、改动对比、人工确认和发布门。

### 4.3 批注修订流程

```text
写作
  -> 选中文本右键创建批注
  -> 右侧批注列表勾选批注
  -> AI 工作台
  -> 按批注生成修订草稿
  -> 检查草稿
  -> 查看改动
  -> 确认写回正文
```

说明：

- 批注是修订依据。
- AI 修订产物先进入草稿，不直接改正文。

### 4.4 设定/章纲提案流程

```text
AI 素材库
  -> 选择系统设定 / 小说设定 / 章纲
  -> 生成提案
  -> 查看提案
  -> 查看改动
  -> 人工采纳
```

说明：

- 当前不提供普通发布按钮直接覆盖设定或章纲。
- 这是为了避免 AI 修改基础资料后影响全书一致性。

### 4.5 自动流水线流程

```text
设置/模型
  -> 确认作品、模型和额度
  -> 自动流水线
  -> 选择章节范围和模式
  -> 保持 dry-run
  -> 创建流水线
  -> 运行队列
  -> 查看每章结果
  -> 需要人工处理的章节交给作者
  -> 需要写回的草稿回到 AI 工作台确认
```

说明：

- 自动流水线用于批量生产和验证。
- 不应替代日常人工编辑。
- 真实工作区建议先 dry-run，沙盒验证通过后再考虑写回。

## 5. 推荐的信息架构原则

1. 写作页只服务作者的正文编辑，不堆模型、成本和发布细节。
2. AI 工作台处理草稿检查、修订、差异和写回确认。
3. AI 素材库只服务设定和章纲，不展示正文目录。
4. 自动流水线只做批量流程和报告，不绕过发布门。
5. 设置/模型承载高级信息，普通作者不必每天进入。
6. 所有写回必须有草稿或版本、校验、改动对比、人工确认和备份。

## 6. 后续优化点

- AI 工作台主路径继续减少对手动草稿编号的依赖，优先通过最近草稿、当前章节和检查状态选择。
- 设置/模型继续拆分为作者可读状态和排错详情，降低一次性信息量。
- 文档与 README 需要在每次入口变更后同步更新，避免旧流程重新污染实现判断。
