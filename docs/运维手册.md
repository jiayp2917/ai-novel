# 小说编辑器运维手册

## 配置文件

- `.env.example`：配置模板，可以复制为 `.env`。
- `key.txt`：本机真实 key 文件，禁止提交、打印、写入报告。
- `config/model_registry.yaml`：模型 provider、model、role 路由注册表。

当前默认 AI 路径为 Agnes AI：`AGNES_BASE_URL=https://apihub.agnes-ai.com/v1`，密钥环境变量为 `AGNES_API_KEY`。`key.txt` 支持读取 `https://apihub.agnes-ai.com/v1 <key>` 形式，并只设置环境变量，不回显明文。

## 工作区与运行态

推荐长期结构：

```text
<repo-root>\                    # 系统代码
<workspace-root>\作品名\         # 小说工作区
```

小说工作区内保存内容源和运行态：

```text
00-系统/
01-设定/
02-正文/
03-章纲/
runtime/
  artifacts/
  backups/
  diffs/
  logs/
  reports/
  app.db
```

系统仍兼容仓库内旧工作区，但新作品建议放到外部工作区。

作品级 skill 也属于小说工作区。通用规则放在系统仓库 `skills/`；某本书的风格规则、审核清单或长期偏好放在外部小说工作区的 `skills/` 目录，不随系统代码提交。

## 启动

普通后端：

```powershell
python -m uvicorn backend.app.main:app --host 127.0.0.1 --port 8000
```

读取本机 `key.txt` 启动：

```powershell
python -m backend.tools.run_backend_with_keys
```

前端：

```powershell
cd frontend
npm run dev
```

## 沙盒手测后端

手动指定测试数据库或运行态目录时，先跑迁移，再启动服务：

```powershell
$env:CONTENT_ROOT='runtime/sandbox_workspace'
$env:APP_DB_PATH='runtime/e2e_runtime/e2e_app.db'
$env:RUNTIME_ROOT='runtime/e2e_runtime'
$env:ENABLE_TEST_SUPPORT='true'
python -m alembic upgrade head
python -m uvicorn backend.app.main:app --host 127.0.0.1 --port 18080
```

也可以直接运行：

```powershell
python -m backend.tools.run_e2e_backend
```

该脚本会创建沙盒工作区并先执行数据库迁移，适合浏览器手测和 E2E 调试。


## 沙盒验证

发布门沙盒：

```powershell
python -m backend.tools.sandbox_publish_smoke
```

流水线沙盒：

```powershell
python -m backend.tools.sandbox_pipeline_smoke --workspace runtime/sandbox_pipeline_workspace --chapters 3 --reset
```

`--reset` 只允许安全沙盒目录，不能用于真实小说工作区。

## 模型与成本报告

模型连通性：

```powershell
python -m backend.tools.probe_roles --roles writer reviewer fixer quick_fix memory long_context outliner structural_fix arbiter --key-file key.txt --force
```

模型使用报告：

```powershell
python -m backend.tools.model_usage_report --out runtime\logs\model_usage_report.md
```

报告里的 token/usage 是本地日志可见下限。真实消耗以供应商控制台为准。

## 沙盒内容写回

本机示例工作区位于 `runtime/sandbox_workspace`。需要把沙盒内容作为本机示例素材时，只同步非 `runtime/` 内容到：

```text
content/settings/
content/outlines/
content/chapters/
```

`content/` 下真实素材和 `content/runtime/` 均被 `.gitignore` 忽略，不随系统代码提交。不要把外部小说工作区中的真实作品当作系统代码提交。

如果从仓库迁出特定作品 skill，只移动到对应作品工作区的 `skills/` 目录，并同步删除仓库默认加载列表中的引用。

## 系统打包

只打系统代码和 `content/` 空模板占位，不包含小说正文、设定、章纲、runtime、key、`.env`：

```powershell
python -m backend.tools.package_system --dry-run
python -m backend.tools.package_system --out <packages-dir>\novel_editor_system.zip
```

打包工具会拒绝把 `00-系统`、`01-设定`、`02-正文`、`03-章纲`、`runtime`、`key.txt`、`.env`、`.env.*` 放入压缩包。
